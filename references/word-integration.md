# التكامل الجراحيّ مع وورد (الطبقة C)

الهدف: إعادة الملف إلى الباحث وقد وُسِمت مواضعُ الإفصاح **داخل الملف نفسه** — تعليقات على المواضع، أو تصحيحات جراحية عند الطلب — **دون تفكيك الملف وإعادة بنائه**، حفاظاً على التنسيق والجداول. نعتمد أدوات مهارة `docx` الجاهزة و`scripts/place_marker.py`.

> اسأل الباحث أولاً: تعليقات فقط (الأأمن، غير مدمّر)، أم تصحيحات جراحية أيضاً؟ لا تعدّل النصّ الأصليّ دون إذنٍ صريح. **ولا تكتب بنفسك بيان إفصاحٍ نيابةً عن الباحث كأنه إقرارٌ منه** — أنت تقترح صياغةً في تعليقٍ يقبلها هو، لا تُنشئ إقراراً باسمه.

## سير العمل الكامل

```bash
DOCX=/mnt/skills/public/docx/scripts     # اختصار

# 1) فكّ الملف وأزل الروابط الرمزية
unzip -q file.docx -d unpacked/ && find unpacked -type l -delete

# 2) ادمج الـ runs كي يصير نصّ موضع الإفصاح قابلاً للإيجاد
python $DOCX/merge_runs.py unpacked/

# 3) لكل ملاحظة: أنشئ تعليقاً (يعيد id) ثم رسّخ علامته حول الموضع
python $DOCX/comment.py unpacked/ "🔵 نقص إفصاح: العائلة كمّية؛ يُستحسن إضافة بيان توافر البيانات."   # → id=0
python scripts/place_marker.py unpacked/word/document.xml --id 0 --target "تحليل النتائج"

python $DOCX/comment.py unpacked/ "🟠 تعارض إفصاحيّ: تصريحٌ بلا تمويل هنا، وذكرُ مموّلٍ في الشكر. يُرجى التوضيح."   # → id=1
python scripts/place_marker.py unpacked/word/document.xml --id 1 --target "لم يتلقَّ البحث"

# 4) أعد البناء
(cd unpacked && rm -f ../out.docx && zip -Xr ../out.docx .)

# 5) تحقّق من السلامة (إلزامي)
python $DOCX/office/validate.py out.docx --original file.docx
```

## ملاحظات حاسمة
- **رتّب: أنشئ التعليق ثم رسّخ علامته فوراً**، ثم انتقل للتالي. الأرقام تُسنَد تصاعدياً.
- **`merge_runs.py` شرطٌ للترسيخ:** إن فشل `place_marker.py`، قصّر الهدف إلى شذرةٍ متصلةٍ مميّزة.
- **العلامات أبناءٌ مباشرون لـ `<w:p>`**، لا داخل `<w:r>`. الأداة تلتزم بذلك.
- **التحقق غير قابل للتخطّي:** أيّ «Unreferenced/Broken reference» يجعل الملف يبدو تالفاً. صحّحه قبل التسليم (`--auto-repair` للشائع).
- **مواضع الإفصاح كثيراً ما تكون في صفحة الإقرار أو الشكر أو حاشيةٍ ختامية**؛ استهدف شذرةً مميّزةً من الموضع نفسه.

## ماذا تكتب في التعليق
اجعل كل تعليق **مكتفياً بذاته وقابلاً للتنفيذ، وضمن الجدار الأخلاقيّ**:
> [شدّة] المشكلة (بند/تعارض) → لماذا (بالمصفوفة/التعارض/التحقق) → الإصلاح المقترح (إضافة/استكمال/توضيح — لا اتهام).

**مثال سليم:** «🔵 نقص إفصاحٍ شكليّ: العائلة كمّية على بشر، ومصفوفة الانطباق تتوقّع بيان الموافقة المستنيرة؛ الملف لا يُظهره في المواضع المعتادة. الإصلاح: أضِف جملةً تذكر أخذ الموافقة وطريقتها. (إن كانت الجامعة لا تشترط لجنة رسمية، فاذكر ذلك)».

**مثال ممنوع:** «الباحث أخفى تضارب المصالح» — هذا اتهامٌ بحقّ شخص، خارج سقف المهارة. الصواب: «🟠 الموضعان يبدوان متعارضين؛ يُرجى من الباحث توضيح العلاقة».

## التصحيحات الجراحية (بإذن فقط)
لاستكمال بيانٍ ناقصٍ (كإضافة جهة اتصالٍ لبيان «عند الطلب») استعمل **التغييرات المتعقَّبة** (Tracked Changes) كي يقبل/يرفض الباحث، لا تعديلاً صامتاً. ولا تكتب أرقاماً أو جهاتٍ من عندك؛ اترك موضعَها للباحث.
